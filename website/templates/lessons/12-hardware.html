Qubit Technologies

Several physical systems can implement qubits, each with trade-offs:

  • Superconducting qubits (IBM, Google, Rigetti): Fast gates (~20ns), limited connectivity, require dilution refrigerators at ~15 mK. Most mature technology.
  • Trapped ions (IonQ, Quantinuum): High fidelity, all-to-all connectivity, slower gates (~ms). Excellent for small circuits.
  • Photonic qubits (Xanadu): Room temperature operation, natural for communication. Challenging for gate-based computation.
  • Neutral atoms (QuEra): Scalable arrays, flexible connectivity. Rapidly advancing.

Hardware Constraints

Real hardware has limitations that ideal simulators don't:

  • Basis gates: Each QPU supports only a few native gates. Other gates must be decomposed into these.
  • Connectivity: Not all qubit pairs can interact directly. SWAP gates must be inserted to route operations between non-adjacent qubits.
  • Gate fidelity: Each gate has a small error probability. Fewer gates = fewer errors.

Transpilation

Transpilation automatically adapts an abstract circuit to run on specific hardware. Goqu's transpiler handles:

  1. Gate decomposition: Convert gates to the target's native basis
  2. Qubit routing: Insert SWAPs for connectivity constraints (SABRE algorithm)
  3. Optimization: Cancel redundant gates, merge rotations, commute gates

Running on Real Hardware

// Transpile for IonQ
compiled, _ := pipeline.Run(ctx, circuit, target.IonQForte, pipeline.LevelFull)

// Submit to IonQ
backend := ionq.New(apiKey)
job, _ := backend.Submit(ctx, &backend.SubmitRequest{
    Circuit: compiled,
    Shots:   1000,
})

// Wait for results
result, _ := backend.Result(ctx, job.ID)
fmt.Println(result.Counts)

// Or use the job manager for multiple backends
mgr := manager.New()
mgr.Register("ionq", ionq.New(ionqKey))
mgr.Register("ibm", ibm.New(ibmKey))
results, _ := mgr.SubmitBatch(ctx, circuit, 1000, "ionq", "ibm")

Exercises

  1. Compare the transpiled circuits for IonQ vs. IBM. Why does IonQ need fewer routing SWAPs? (Hint: connectivity)
  2. If a QPU has 99.9% single-qubit gate fidelity and 99% two-qubit gate fidelity, what is the expected fidelity of a circuit with 10 single-qubit and 5 two-qubit gates?
  3. Why might you choose IonQ (fewer qubits, all-to-all) over IBM (more qubits, limited connectivity) for certain algorithms?
