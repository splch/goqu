7. Grover's Search Algorithm

{{.Lesson.Desc}}

Learning Objectives

  • Understand the unstructured search problem

The Search Problem

Given an unsorted database of N items, find the one marked item. Classically, this requires O(N) queries. Grover's algorithm does it in O(√N) — a quadratic speedup.

How It Works

Grover's algorithm uses two key operations repeated approximately π/4 · √N times:

  1. Oracle: Marks the target state by flipping its phase (|x⟩ → −|x⟩ for the marked x)
  2. Diffusion (Grover operator): Reflects all amplitudes about their mean, amplifying the marked state's probability

Each iteration increases the probability of measuring the marked state. After the optimal number of iterations, the probability is nearly 1.

Goqu Code

// Search for state |101⟩ among 3-qubit states
oracle := func(b *builder.Builder, qubits []int) {
    // Phase-flip |101⟩
    b.X(qubits[1]) // Flip qubit 1 (should be 0 in target)
    b.CCZ(qubits[0], qubits[1], qubits[2])
    b.X(qubits[1]) // Undo flip
}

res, _ := grover.Run(ctx, grover.Config{
    NumQubits:    3,
    Oracle:       oracle,
    NumSolutions: 1,
    Shots:        1024,
})
fmt.Println("Found:", res.TopResult)

Exercises

  1. For 3 qubits (8 states), the optimal number of Grover iterations is about 2. Verify: π/4 · √8 ≈ 2.2. What happens if you use too many iterations?
  2. Grover's gives a quadratic speedup. For a database of 10⁶ items, how many classical queries are needed vs. quantum?
  3. Can Grover's algorithm find multiple marked items? How does the iteration count change?
