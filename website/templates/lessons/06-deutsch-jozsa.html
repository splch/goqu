6. The Deutsch-Jozsa Algorithm

{{.Lesson.Desc}}

Learning Objectives

  • Understand the oracle model of quantum computation
  • Learn the Deutsch-Jozsa algorithm and why it matters
  • See the first example of quantum advantage over classical computing
  • Run the algorithm with different oracles

The Problem

Given a black-box function f: {0,1}ⁿ → {0,1}, we are promised that f is either:

  • Constant: f(x) = 0 for all x, or f(x) = 1 for all x
  • Balanced: f outputs 0 for exactly half of all inputs and 1 for the other half

The task: determine which case it is.

Classical vs. Quantum

Classically, in the worst case you need to evaluate f on 2ⁿ⁻¹ + 1 inputs — more than half of all possibilities — before you can be certain.

Quantumly, the Deutsch-Jozsa algorithm determines the answer with just one evaluation of f, using superposition and interference.

How It Works

  1. Prepare n input qubits in |0⟩ and one ancilla qubit in |1⟩
  2. Apply Hadamard gates to all qubits, creating a uniform superposition
  3. Apply the oracle: the function f is encoded as a quantum gate
  4. Apply Hadamard gates to the input qubits again
  5. Measure the input qubits: if all are |0⟩, f is constant; otherwise, f is balanced

Goqu Code

// Run Deutsch-Jozsa with a balanced oracle
oracle := func(b *builder.Builder, inputs []int, ancilla int) {
    b.CNOT(inputs[0], ancilla) // f(x) = x₀
}

result, _ := textbook.DeutschJozsa(ctx, textbook.DJConfig{
    NumQubits: 3,
    Oracle:    oracle,
    Shots:     1024,
})
fmt.Println("Is constant:", result.IsConstant)

Exercises

  1. For a constant oracle, what measurement outcome do you expect and why?
  2. This algorithm gives an exponential speedup. For 100-bit inputs, how many evaluations does a classical computer need in the worst case? How many does the quantum algorithm need?
  3. The Deutsch-Jozsa algorithm works because of destructive interference. Can you trace through the math for 1 qubit to see why?
