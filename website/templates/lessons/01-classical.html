1. Classical Computing Review

Learning Objectives

  • Understand classical bits and binary representation
  • Review Boolean logic gates (AND, OR, NOT, XOR)
  • See why classical computers have fundamental limitations
  • Motivate the need for quantum computing

Classical Bits

All classical computation is built on bits — the smallest unit of information. A bit is always in one of two states: 0 or 1. Every number, image, and program on your computer is encoded as a sequence of bits.

With n bits, you can represent 2n different values. For example, 8 bits (a byte) can represent 256 values (0 through 255).

Logic Gates

Classical computers process bits using logic gates — simple operations that transform input bits into output bits:

  • NOT: Flips a bit. 0 → 1, 1 → 0
  • AND: Output is 1 only if both inputs are 1
  • OR: Output is 1 if at least one input is 1
  • XOR: Output is 1 if inputs are different

Every computation — from adding numbers to rendering graphics — is built from combinations of these simple gates.

Reversibility

An important distinction: most classical gates are irreversible. An AND gate with output 0 could have had inputs (0,0), (0,1), or (1,0) — you can't tell which. Information is lost.

Quantum gates, as we'll see, are always reversible (unitary). This is a fundamental requirement of quantum mechanics and has deep implications for computation.

Why Quantum?

Classical computers are incredibly powerful, but some problems remain intractable:

  • Factoring large numbers: The security of RSA encryption relies on this being hard. The best classical algorithm takes exponential time.
  • Simulating quantum systems: A system of n quantum particles needs 2n classical numbers to describe its state.
  • Searching unsorted data: Classically requires checking every item. Quantum computers can do it with the square root of the checks.

Quantum computing exploits the principles of quantum mechanics — superposition, entanglement, and interference — to solve these problems more efficiently.

Exercises

  1. How many bits would you need to represent all possible states of a chess board? Think about what information you need to encode.
  2. Why is the AND gate irreversible? Can you design a reversible version? (Hint: look up the Toffoli gate)
  3. If a classical computer takes 2128 operations to break a code, and can do 1012 operations per second, how long would it take?
