8. Quantum Fourier Transform & Phase Estimation

{{.Lesson.Desc}}

Learning Objectives

  • Understand the Quantum Fourier Transform (QFT)

  • See QPE estimate eigenvalue phases of unitary operators

Quantum Fourier Transform

The QFT transforms between the computational basis and the Fourier basis. For n qubits, it maps:

QFT|j⟩ = (1/√N) Σₖ e^(2πijk/N) |k⟩

This is the quantum analogue of the classical Discrete Fourier Transform, but it operates on 2ⁿ amplitudes using only O(n²) gates — an exponential speedup over the classical O(N log N).

Phase Estimation

QPE solves a fundamental problem: given a unitary operator U and its eigenstate |ψ⟩ where U|ψ⟩ = e^(2πiφ)|ψ⟩, estimate the phase φ.

The algorithm uses:

  1. A phase register of t ancilla qubits (determines precision: φ is estimated to t bits)
  2. A target register holding the eigenstate
  3. Controlled-U^(2^k) operations for phase kickback
  4. The inverse QFT to convert phase information into a measurable bit string

Goqu Code

// Estimate the phase of a Phase gate
phaseGate := gate.Phase(2 * math.Pi * 0.25) // φ = 0.25

eigenPrep, _ := builder.New("eigen", 1).X(0).Build()

res, _ := qpe.Run(ctx, qpe.Config{
    Unitary:      phaseGate,
    NumPhaseBits: 3,
    EigenState:   eigenPrep,
    Shots:        1024,
})
fmt.Printf("Estimated phase: %.4f\n", res.Phase)

Exercises

  1. Try φ = 0.25 with 3 precision bits. You should get an exact result. Why? (Hint: 0.25 = 2/8, and 8 = 2³)
  2. Now try φ = 1/3 ≈ 0.333. The result won't be exact. Why not? How can you improve accuracy?
  3. QPE is the core subroutine in Shor's algorithm (next lesson). What "phase" does Shor's algorithm estimate?
