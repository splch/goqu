9. Shor's Factoring Algorithm

{{.Lesson.Desc}}

Learning Objectives

  • Understand why factoring is important (RSA encryption)
  • Learn how Shor's algorithm reduces factoring to period-finding
  • See the connection between QPE and order-finding
  • Observe a period-finding circuit in action

Why Factoring Matters

RSA encryption, used to secure internet communications, relies on the assumption that factoring large numbers (e.g., 2048-bit) is computationally infeasible. The best classical algorithm takes sub-exponential time. Shor's algorithm factors in polynomial time, threatening RSA if large quantum computers are built.

From Factoring to Period-Finding

Shor's key insight: factoring N reduces to finding the period of the function f(x) = aˣ mod N for a random base a.

  1. Pick random a < N with gcd(a, N) = 1
  2. Find the period r: the smallest r such that aʳ ≡ 1 (mod N)
  3. If r is even, compute gcd(a^(r/2) ± 1, N) to find factors

Step 2 is where the quantum computer comes in: QPE can find the period exponentially faster than any known classical algorithm.

Goqu Code

// Factor 15 using Shor's algorithm
res, _ := shor.Run(ctx, shor.Config{
    N:     15,
    Shots: 1024,
})
fmt.Printf("Factors: %d × %d\n", res.Factors[0], res.Factors[1])
fmt.Printf("Period: %d, Base: %d\n", res.Period, res.Base)

Complexity

Classical (Number Field Sieve): exp(O(n^(1/3) · (log n)^(2/3)))
Quantum (Shor's):              O(n² · log n · log log n)

For a 2048-bit number:
  Classical: ~10^30 operations (infeasible)
  Quantum:   ~10^10 operations (feasible with error correction)

Exercises

  1. The measurement results should peak at multiples of 2ⁿ/r. For N=15 and a=2, the period is r=4. With 4 phase qubits (2⁴=16), where should the peaks be?
  2. Why does the algorithm fail if r is odd? What happens if a^(r/2) ≡ −1 (mod N)?
  3. If quantum computers can break RSA, what alternatives exist? (Research: lattice-based cryptography, post-quantum cryptography)
